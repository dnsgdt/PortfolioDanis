Épreuve E6 – Situation professionnelle

1. Contexte

Dans le cadre de ma formation en BTS SIO option SISR, j'ai réalisé le projet BELLETABLE. L'objectif était de concevoir et déployer une solution complète de gestion de parc informatique permettant le suivi du matériel, des utilisateurs, des tickets d'incidents et des consommables.

Le projet a été réalisé dans un environnement virtualisé via Proxmox. Une machine virtuelle Debian 12 a été déployée afin d'héberger GLPI, outil open-source de gestion de parc informatique.

Cette situation a été réalisé avec Yohan DO-CAO.

2. Architecture technique mise en place

  • Hyperviseur : Proxmox VE
  • Système serveur : Debian 12
  • Serveur Web : Apache2
  • Base de données : MariaDB
  • Langage supporté : PHP
  • Application : GLPI
  • Authentification : LDAP

3. Installation et configuration

  • Installation Apache : apt install apache2
  • Installation MariaDB : apt install mariadb-server
  • Installation PHP et modules requis
  • Création base de données sécurisée
  • Configuration droits SQL
  • Déploiement GLPI dans /var/www/html

4. Configuration LDAP

Mise en place de l'authentification centralisée via LDAP. Un problème de bind a été identifié puis corrigé après analyse des paramètres DN et identifiants.

5. Gestion des profils

  • Administrateur
  • Technicien
  • Utilisateur
  • Observateur

6. Résultats

  • GLPI opérationnel
  • LDAP fonctionnel
  • Système de tickets validé
  • Gestion complète du parc

7. Analyse personnelle

Ce projet m'a permis de consolider mes compétences en administration système, virtualisation et gestion d'infrastructure.

Production – Projet BELLETABLE

1. Présentation de la production

La production réalisée dans le cadre du projet BELLETABLE consiste au déploiement complet d'une solution de gestion de parc informatique basée sur GLPI. L'objectif était de fournir une infrastructure fonctionnelle, sécurisée et administrable.

2. Infrastructure mise en place

  • Création d'une machine virtuelle sous Proxmox
  • Installation de Debian 12
  • Configuration réseau (IP statique)
  • Installation Apache2
  • Installation MariaDB
  • Installation PHP et modules nécessaires
  • Déploiement et configuration de GLPI

3. Configuration base de données

Création d'une base de données dédiée à GLPI. Configuration d'un utilisateur SQL sécurisé avec droits restreints. Sécurisation via mysql_secure_installation.

Capture configuration base de données

4. Paramétrage fonctionnel dans GLPI

  • Création des entités
  • Création des profils utilisateurs
  • Paramétrage des droits d'accès
  • Configuration du système de tickets
  • Ajout des catégories d'incidents

5. Intégration LDAP

Mise en place d'une authentification centralisée via LDAP. Configuration du DN, du serveur LDAP et des filtres utilisateurs. Tests de connexion et correction d'erreurs de bind.

6. Gestion du parc

  • Ajout des postes de travail
  • Ajout des serveurs
  • Ajout des équipements réseau
  • Gestion des consommables
  • Attribution des équipements aux utilisateurs

Capture gestion du parc

7. Sécurisation et sauvegarde

Réalisation d'un dump de la base MariaDB via mysqldump. Sauvegarde du fichier glpi_backup.sql. Vérification de l'intégrité des données.

8. Résultat final

L'environnement est entièrement fonctionnel. Les utilisateurs peuvent se connecter via LDAP, ouvrir des tickets et consulter le parc informatique. La solution répond aux besoins de gestion et de suivi.

9. Compétences mobilisées

  • Administration système Linux
  • Virtualisation
  • Gestion base de données
  • Configuration réseau
  • Gestion des droits utilisateurs
  • Diagnostic et résolution d'incidents

Veille technologique

Cybersécurité & Intelligence Artificielle

Comment l'IA transforme le paysage des menaces informatiques — et comment les défenses s'adaptent en temps réel pour protéger les systèmes d'information.

Menace Janvier 2025

L'IA au service des attaquants : phishing de nouvelle génération

Les cybercriminels utilisent désormais des LLM (grands modèles de langage) pour générer des emails de phishing ultra-personnalisés, sans fautes, dans n'importe quelle langue. Ces attaques contournent les filtres classiques basés sur les signatures de texte. Le taux de clics sur ces messages est significativement plus élevé que sur les campagnes traditionnelles.

Filtrage email renforcé Sensibilisation utilisateurs Analyse comportementale
→ Source : ANSSI / CERT-FR
IA Défensive Février 2025

EDR et XDR : la détection comportementale dopée à l'IA

Les solutions EDR (Endpoint Detection & Response) et XDR (Extended Detection & Response) intègrent désormais des modèles d'apprentissage automatique capables de détecter des comportements anormaux en quelques secondes. CrowdStrike Falcon ou Microsoft Defender for Endpoint analysent des millions d'événements en temps réel pour identifier des menaces zero-day, même inconnues des bases de signatures.

Réduction du temps de détection Réponse automatisée Zero-day coverage
→ Source : CrowdStrike
Deepfake Mars 2025

Deepfakes vocaux et vidéo : la fraude au président 2.0

Des deepfakes audio et vidéo sont utilisés dans des attaques de type "fraude au président" : des employés reçoivent des appels ou des visioconférences avec un faux dirigeant leur demandant des virements urgents. Des entreprises ont subi des pertes de plusieurs millions d'euros. L'authentification multi-facteurs et les procédures de validation hors-bande deviennent essentielles pour contrer ces scénarios.

MFA renforcé Procédures de validation Formation dirigeants
→ Source : IBM Security
Réglementation Avril 2025

NIS2 et IA Act : le nouveau cadre réglementaire européen

La directive NIS2 (Network and Information Security) élargit les obligations de cybersécurité à de nouveaux secteurs, dont la culture et les institutions publiques. Combinée à l'AI Act européen, elle impose aux organisations d'auditer leurs systèmes IA utilisés en sécurité. Les DSI doivent désormais documenter leurs outils IA et prouver leur conformité.

Audit des systèmes IA Documentation obligatoire Conformité RGPD/NIS2
→ Source : Microsoft Security
Infrastructure Avril 2025

Zero Trust Architecture : vers un modèle "ne jamais faire confiance"

Le modèle Zero Trust repose sur le principe qu'aucun utilisateur, appareil ou flux réseau n'est digne de confiance par défaut, même à l'intérieur du réseau interne. Cette approche, popularisée par Microsoft et Cisco, implique une vérification continue des identités, une micro-segmentation des réseaux et une authentification conditionnelle basée sur le contexte (localisation, heure, comportement).
